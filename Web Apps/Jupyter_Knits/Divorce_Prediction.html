Comparing Bayes, Logistic, and KNN Classifiers on Marriage Dataset:¶

In this project, I will use several classifiers to analyze the marriages dataset for divorce prediction.

This dataset is about participants who completed the personal information form and a divorce predictors scale.

The data is a modified version of the publicly available at https://archive.ics.uci.edu/ml/datasets/Divorce+Predictors+data+set (by injecting noise so you will not get the exactly same results as on UCI website). There are 170 participants and 54 attributes (or predictor variables) that are all real-valued. The last column of the CSV file is label $y$ (1 means divorce, 0 means no divorce). Each column is for one feature (predictor variable), and each row is a sample (participant). A detailed explanation for each feature (predictor variable) can be found at the website link above. Our goal is to build a classifier using training data, such that given a test sample, we can classify (or essentially predict) whether its label is 0 (no divorce) or 1 (divorce).

We will build three classifiers using Naive Bayes, Logistic Regression, KNN. Use the first $80\%$ data for training and the remaining $20\%$ for testing.

We begin by loading up the dataset and any relevant packages and formatting the data properly as demonstrated in the code cell below:

data_train,data_test,labels_train,labels_test = train_test_split(marriage_data,marriage_labels,test_size=.2)

With our data and labels as well as packages loaded properly we move onto fitting each of the three classifiers beginning with logistic regression.

Logistic Regression Accuracy Over Many Random Seeds:¶

After formatting our data properly, we can split it into two sets one for model training and a left out set for model testing. We will use this splitting method in training and testing each of the models below. Additionally, because the splitting method involves randomly choosing observations, we will run 100 iterations of each model with varying but comparable random number generator seeds to better understand the model's adaptability to initial conditions. Now we input the training data into our logistic regression model then evaluate the accuracy of the model via the testing data. The result of each of these randomly initialized training and testing set iterations is displayed below with an accompanying line showing the average rate across all iterations.

In [3]:
def logistic_seeds():
    logistic_regression_accuracy_list = []
    for i in range(0,100):
        data_train,data_test,labels_train,labels_test = train_test_split(marriage_data,marriage_labels,test_size=.2,random_state=i)
        logistic_function = LogisticRegression(random_state=i).fit(data_train,labels_train)
        logistic_preds = logistic_function.predict(data_test)
        logistic_acc = np.sum(labels_test == logistic_preds)/labels_test.shape
        logistic_regression_accuracy_list.append(logistic_acc[0])
    return logistic_regression_accuracy_list
log_reg_seeded = logistic_seeds()
print('Maximum Test Accuracy = %s'%np.max(log_reg_seeded))
plt.plot(log_reg_seeded)
log_reg_average = np.average(log_reg_seeded)
average_list_log_1 = [log_reg_average]*100
plt.plot(np.array(average_list_log_1))
plt.title('Logistic Regression Accuracy Over Various Initializations')
plt.xlabel('Random Seed')
plt.ylabel('Model Test Accuracy')
plt.show()

Maximum Test Accuracy = 1.0

As we can see from the chart output above, our logistic regression model does quite well generally at predicating divorces. We can see the chart shows some variability based upon the random seed used in the splitting step of our implementation, but overall the differences from iteration to iteration is fairly minimal suggesting we are somewhat robust to initial conditions with this dataset. Over all 100 different initializations our model comes out with an average accuracy rate of ~98%, which I found surprisingly high.

The strong performance of PCA also adds evidence to the hypothesis that our dataset is highly linearly separable as PCA excels in situations where variables share linear separability. Comparing Models:¶

Given all of our results shown above, I would if forced to choose between the models pick the logistic regression approach as the best model for a couple of reasons. Firstly, the logistic regression model with PCA reduction achieved the highest average percentage test accuracy tying with KNN with PCA reduction. The reason for choosing the logistic model over Knn in this scenario is twofold, the knn algorithm requires the tuning parameter of number of neighbors which alters this testing accuracy value and adds more complexity in properly applying the model. Additionally, the knn model as demonstrated in the decision boundary visualizations has a far more irregular seperation plane which although benefits training accuracy can hinder what important which is truly test accuracy.
